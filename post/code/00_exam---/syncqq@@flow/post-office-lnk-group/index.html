todo-done-as-target

郵局連結群組管理工具 (Post Office Link Group Manager)

這是一個用於管理和組織 Windows 捷徑檔案的 Python 工具，特別適用於整理考試資料、學習資源和各種文件連結。

功能特色

  • 🔗 自動建立捷徑: 從路徑清單自動建立 Windows .lnk 捷徑檔案
  • 📁 智能分類: 自動為每個捷徑建立編號前綴和對應資料夾
  • 📊 HTML 報表: 生成包含所有連結的 HTML 表格報表
  • 🗂️ 多種輸出格式: 支援不同主題和分類的輸出目錄
  • 📝 詳細日誌: 完整的錯誤記錄和處理狀態追蹤

專案結構

app/
├── app.py                    # 主要程式檔案
├── path.txt                  # 路徑清單檔案
├── requirements.txt          # Python 依賴套件
├── alllnk/                   # 原始連結檔案
├── output/                   # 預設輸出目錄
├── output-chinese/           # 中文內容輸出
├── output-network/           # 網路相關輸出
├── output-post/              # 郵局相關輸出
├── output-postoffice/        # 郵局考試輸出
├── output-program-normal/    # 程式設計一般輸出
├── output-taipower/          # 台電相關輸出
├── output-ttt/               # TTT 相關輸出
├── output-twport/            # 台灣港務輸出
├── output-v1/                # 版本 1 輸出
└── 00_version/               # 版本歷史檔案

安裝需求

系統需求

  • Windows 作業系統
  • Python 3.6 或更高版本

依賴套件

pip install -r requirements.txt

主要依賴：

  • pywin32==306 - Windows COM 物件支援

使用方法

1. 準備路徑清單

在 path.txt 檔案中輸入要建立捷徑的檔案或資料夾路徑，每行一個路徑：

D:\path\to\file1.txt
D:\path\to\folder1
D:\path\to\file2.pdf

2. 執行程式

python app.py

3. 查看結果

程式執行完成後，會在 output 目錄中生成：

  • 編號資料夾（01_、02_、03_…）
  • 每個資料夾包含對應的 .lnk 捷徑檔案
  • output_combinedTable.html - 包含所有連結的 HTML 報表

輸出格式

資料夾結構

output/
├── 01_檔案名稱/
│   └── 01_檔案名稱.lnk
├── 02_檔案名稱/
│   └── 02_檔案名稱.lnk
└── output_combinedTable.html

HTML 報表

生成的 HTML 檔案包含：

  • 所有捷徑的表格清單
  • 可點擊的連結
  • 檔案路徑資訊
  • 美觀的網頁介面

特殊功能

多主題支援

專案支援多種不同的輸出主題：

  • 中文內容: output-chinese/
  • 網路技術: output-network/
  • 郵局考試: output-postoffice/
  • 程式設計: output-program-normal/
  • 台電考試: output-taipower/
  • 台灣港務: output-twport/

批次處理

  • 支援大量檔案路徑的批次處理
  • 自動錯誤處理和日誌記錄
  • 進度顯示和統計資訊

錯誤處理

程式會自動記錄所有錯誤到日誌檔案：

  • 檔案名稱格式：error_log_YYYYMMDD_HHMMSS.txt
  • 包含詳細的錯誤訊息和時間戳記
  • 不存在的路徑會被記錄但不會中斷程式執行

版本歷史

查看 00_version/ 目錄了解版本演進：

  • v1_path-to-lnk.py - 第一版實作
  • v2_sig-path-log;wait-number-prefix-folder.py - 第二版改進

注意事項

  1. 路徑格式: 確保 path.txt 中的路徑使用正確的 Windows 路徑格式
  2. 權限: 程式需要寫入權限來建立捷徑檔案
  3. 編碼: 路徑檔案使用 UTF-8 編碼
  4. 清理: 程式會自動清理臨時檔案

故障排除

常見問題

Q: 捷徑無法建立 A: 檢查目標路徑是否存在，以及是否有足夠的權限

Q: 中文路徑顯示異常 A: 確保 path.txt 使用 UTF-8 編碼儲存

Q: 程式執行緩慢 A: 大量檔案處理時屬正常現象，可查看控制台輸出了解進度
