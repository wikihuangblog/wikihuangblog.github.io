Markdown 文件分割工具

這是一個用於將大型 Markdown 文件按章節分割成小文件的 Python 工具，支援自動添加 YAML front matter、簡體中文轉繁體中文、圖片處理等功能。

功能特色

  • 📄 智能分割：根據 ### 標題自動分割 Markdown 文件
  • 🏷️ 自動分類：根據內容自動判斷是否為程式碼類型並添加相應標籤
  • 🔤 繁簡轉換：自動將簡體中文轉換為繁體中文
  • 🖼️ 圖片處理：自動複製圖片到對應資料夾並更新路徑
  • 📝 YAML Front Matter：自動生成包含標題、分類、時間等資訊的 YAML 前置資料
  • 📁 結構化輸出：按編號創建子資料夾，便於管理

安裝需求

Python 套件

pip install opencc

系統需求

  • Python 3.6+
  • Windows 系統（用於執行 run.bat）

使用方法

方法一：使用批次檔（推薦）

run.bat your_file.md

方法二：直接使用 Python

python app.py your_file.md

輸出結構

工具會在當前目錄創建 split_sections 資料夾，結構如下：

split_sections/
├── 00/
│   ├── 00_章節標題__________{codeQQ}_____.QQ.md
│   └── pic/  (如果有圖片)
│       └── image1.png
├── 01/
│   ├── 01_另一個章節__________{}_____.QQ.md
│   └── pic/
└── ...

檔案命名規則

  • 格式：{編號:02d}_{標題}__________{{分類}}_____.QQ.md
  • 編號：從 00 開始的兩位數編號
  • 標題：從章節標題提取，移除特殊字元但保留 [v] 等標記
  • 分類：根據內容自動判斷
    • codeQQ：包含程式碼區塊（java、python、c、csharp、javascript、vb、html）
    • 空字串：一般內容

YAML Front Matter 格式

每個分割後的文件會自動添加以下 YAML 前置資料：

---
title: 章節標題
category: codeQQ  # 或空字串
tags:
  - "codeQQ"  # 或空字串
description:
keyword:
  - "codeQQ"  # 或空字串
datetime: 2024-01-15 14:30
importanceRank:
draft: "true"
published: "false"
onlinelink:"_____"
frequency: "_____"
difficultLevel: "_____"
---

程式碼檢測

工具會自動檢測以下程式語言：

  • Java (java)
  • Python (python)
  • C (c)
  • C# (csharp)
  • JavaScript (javascript)
  • VB (vb)
  • HTML (html)

圖片處理

  • 自動檢測 Markdown 中的圖片連結
  • 將圖片複製到對應章節的 pic 資料夾
  • 更新 Markdown 中的圖片路徑為相對路徑
  • 支援常見圖片格式

特殊標記處理

  • 保留 [v]、[v+]、[v-] 等特殊標記
  • 移除其他可能導致檔案名稱問題的字元
  • 確保檔案名稱在 Windows 系統中有效

錯誤處理

  • 檢查輸入檔案是否存在
  • 處理檔案讀取錯誤
  • 顯示處理進度和錯誤訊息
  • 跳過無效的圖片路徑並顯示警告

範例

假設您有一個名為 big_document.md 的文件：

# 大文件

### 第一章：基礎概念
這是第一章的內容...

### 第二章：程式碼範例
```python
print("Hello World")

第三章：總結

這是總結內容…


執行 `run.bat big_document.md` 後，會生成：

- `split_sections/00/00_第一章：基礎概念__________{}_____.QQ.md`
- `split_sections/01/01_第二章：程式碼範例__________{codeQQ}_____.QQ.md`
- `split_sections/02/02_第三章：總結__________{}_____.QQ.md`

## 注意事項

1. 確保輸入的 Markdown 文件使用 `### ` 作為章節標題
2. 圖片路徑必須是相對路徑且圖片文件存在
3. 工具會覆蓋已存在的輸出目錄
4. 建議在處理前備份原始文件
