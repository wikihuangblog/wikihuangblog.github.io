todo-done-as-target

考試題目標籤與簡答整合工具

這是一個用於處理考試題目文件的 Python 工具，主要功能是從文件路徑中提取標籤和章節信息，並將對應的簡答內容整合到原始文件中。

功能概述

本工具主要執行以下操作：

  1. 提取文件路徑標籤：從文件路徑中的 {標籤} 格式提取標籤信息
  2. 提取章節信息：從文件路徑中提取 Ch 開頭的章節編號
  3. 整合簡答內容：根據題目編號匹配對應的簡答文件，並將內容轉換為 Markdown 格式
  4. 添加提示信息：在文件末尾添加標準化的學習提示

項目結構

app/
├── app.py                    # 主程序文件
├── target.txt               # 包含待處理文件路徑的文本文件
├── 112-little-ans/          # 簡答文件目錄（HTML格式）
│   ├── 1.html
│   ├── 2.html
│   └── ...
├── output/                  # 輸出文件目錄
│   ├── 01.html
│   ├── 02.html
│   └── ...
├── issue.markdown           # 項目需求說明
├── append-tag.md           # 標籤追加說明
└── README.md               # 本說明文件

核心功能

1. 標籤提取

從文件路徑中提取 {標籤} 格式的標籤，例如：

  • {easyeasy} → 提取 “easyeasy”
  • {marknote} → 提取 “marknote”

2. 章節信息提取

從文件路徑中提取章節編號，例如：

  • Ch01-3資料表示法 → 提取 “Ch01-3資料表示法”

3. 簡答內容整合

  • 根據題目編號匹配對應的簡答文件
  • 將 HTML 格式的簡答轉換為 Markdown 格式
  • 在原始文件末尾添加簡答內容

4. HTML 轉 Markdown 轉換

支援以下 HTML 標籤的轉換：

  • <br> → 換行符
  • <strong> → **粗體**
  • <li> → - 列表項
  • <ul> → 移除標籤

使用方法

1. 準備文件

確保以下文件存在：

  • target.txt：包含待處理的 Markdown 文件路徑列表
  • 112-little-ans/ 目錄：包含對應的簡答 HTML 文件
  • output/ 目錄：包含對應的輸出文件

2. 運行程序

python app.py

3. 處理流程

程序會自動：

  1. 讀取 target.txt 中的所有文件路徑
  2. 對每個 .md 文件執行以下操作：
    • 提取題目編號
    • 查找對應的簡答文件
    • 轉換 HTML 為 Markdown
    • 提取標籤和章節信息
    • 將所有信息追加到文件末尾

文件格式要求

target.txt 格式

每行一個文件路徑，例如：

D:\path\to\file\01_題目內容.QQ.lvv.{easyeasy}.md
D:\path\to\file\02_題目內容.QQ.lvv.{marknote}.md

簡答文件命名規則

  • 112-little-ans/题目-1.html
  • 112-little-ans/题目-2.html
  • 112-little-ans/题目15.html（無連字符格式）

輸出文件命名規則

  • output/01.html
  • output/02.html
  • output/15.html

輸出格式

處理後的文件會在末尾添加以下內容：

-----

-----

### simple ans

[轉換後的簡答內容]




[提取的標籤]



[章節信息]



請你根據這個主題呢，把這裡面的內容說得更加詳細。 這個主題,考試的時候會出什麼樣的公式？ 然後常常出的考題類型。 請你用老師的角度來解釋。 然後模擬考試思考題目作答的流程

錯誤處理

程序包含以下錯誤處理機制：

  • 文件不存在時顯示相應錯誤信息
  • 無法提取題號時跳過該文件
  • 找不到對應簡答文件時顯示警告信息

技術特點

  • 正則表達式匹配：使用正則表達式精確提取文件路徑中的各種信息
  • HTML 解析：將 HTML 格式的簡答轉換為易讀的 Markdown 格式
  • 批量處理：支援批量處理多個文件
  • 錯誤容錯：具備完善的錯誤處理機制

注意事項

  1. 確保所有文件路徑使用正確的編碼（UTF-8）
  2. 簡答文件必須存在於 112-little-ans/ 目錄中
  3. 輸出文件必須存在於 output/ 目錄中
  4. 文件路徑中的題目編號必須與簡答文件名中的編號對應

開發環境

  • Python 3.x
  • 標準庫：re（正則表達式）
