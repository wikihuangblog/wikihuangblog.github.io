批量文件處理工具

這是一個用於批量處理文件的 Python 應用程序，主要功能包括處理 HTML 文件和 .lnk 快捷方式文件。

功能特色

1. HTML 文件處理器 (app.py)

  • 自動掃描指定資料夾中的 HTML 文件
  • 從 HTML 文件中提取 URL 鏈接
  • 按順序打開每個 URL（使用瀏覽器）
  • 處理完成後自動移動文件到完成資料夾
  • 智能延遲計算，基於資料夾名稱中的數字範圍
  • 顯示預估完成時間

2. LNK 文件處理器 (v1_app-open-by-lnk.py)

  • 掃描指定資料夾中的 .lnk 快捷方式文件
  • 自動打開每個快捷方式指向的應用程序
  • 記錄處理日誌
  • 處理完成後移動文件到完成資料夾
  • 支援多層級目錄掃描

系統需求

  • Python 3.6+
  • Windows 作業系統（使用 win32com 模組）
  • 以下 Python 套件：
    • beautifulsoup4
    • pywin32

安裝依賴

pip install beautifulsoup4 pywin32

使用方法

HTML 文件處理

python app.py <目標資料夾路徑>

範例：

python app.py "C:\Users\YourName\Documents\html_files"

功能說明：

  • 程式會掃描指定資料夾中的所有 HTML 文件
  • 從每個 HTML 文件中提取第一個符合條件的 URL
  • 根據資料夾名稱中的數字範圍計算延遲時間
  • 依序打開每個 URL，並在處理完成後移動文件

LNK 文件處理

python v1_app-open-by-lnk.py <目標資料夾路徑>

範例：

python v1_app-open-by-lnk.py "C:\Users\YourName\Desktop\shortcuts"

功能說明：

  • 掃描指定資料夾中的 .lnk 快捷方式文件
  • 自動打開每個快捷方式
  • 記錄處理日誌到 lnk_files_log.txt
  • 處理完成後移動文件到 00_finish_folder 子資料夾

資料夾結構

程式會自動創建以下資料夾結構：

目標資料夾/
├── 原始文件...
├── 00_finish_folder/          # 處理完成的文件會移動到這裡
│   ├── processed_file1.html
│   ├── processed_file2.lnk
│   └── ...
└── lnk_files_log.txt          # LNK 處理器的日誌文件（僅 LNK 處理器）

重要注意事項

HTML 處理器

  • 延遲時間計算：延遲 = 4秒 × 資料夾名稱中間值 + 5秒
  • 資料夾名稱應包含數字範圍格式，如：100-200
  • 程式會顯示預估完成時間
  • 使用單線程處理，確保順序執行

LNK 處理器

  • 每次處理間隔 200 秒（可調整）
  • 支援最大深度為 1 的子目錄掃描
  • 使用 Windows COM 物件來讀取快捷方式
  • 自動記錄所有處理的文件

錯誤處理

  • 如果目標資料夾不存在，程式會顯示錯誤訊息
  • 如果 HTML 文件中沒有找到符合條件的鏈接，會跳過該文件
  • 如果快捷方式文件損壞，會記錄錯誤並繼續處理下一個文件

自定義設定

修改延遲時間

在 app.py 中修改 self.eachWaitSecond 變數：

self.eachWaitSecond = 4  # 改為您想要的秒數

修改 LNK 處理間隔

在 v1_app-open-by-lnk.py 中修改 time.sleep(200) 的值：

time.sleep(200)  # 改為您想要的秒數
