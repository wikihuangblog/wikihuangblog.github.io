HTML 鏈接批量處理工具

這是一個用於批量處理 HTML 文件並自動打開其中鏈接的 Python 工具。

功能描述

此工具會：

  1. 掃描指定資料夾中的所有 HTML 文件
  2. 從每個 HTML 文件中提取第一個符合條件的鏈接（li.classroom-toc-item > a）
  3. 自動在瀏覽器中打開這些鏈接
  4. 根據資料夾名稱計算延遲時間
  5. 處理完成後將 HTML 文件移動到完成資料夾

系統需求

  • Python 3.6+
  • 以下 Python 套件：
    • beautifulsoup4
    • lxml (可選，用於更快的 HTML 解析)

安裝依賴

pip install beautifulsoup4 lxml

使用方法

方法一：使用 Python 腳本

python app.py <目標資料夾路徑>

方法二：使用批次檔

  1. 編輯 go.bat 文件，修改 target_folder 變數為您的目標資料夾路徑
  2. 雙擊執行 go.bat

參數說明

  • target_folder: 包含 HTML 文件的目標資料夾路徑

工作原理

  1. 鏈接提取: 從每個 HTML 文件中查找 li.classroom-toc-item > a 選擇器對應的鏈接
  2. URL 處理: 提取鏈接的基礎 URL（去除文件名部分）
  3. 延遲計算: 根據資料夾名稱中的數字範圍計算延遲時間
    • 例如：資料夾名為 0-9，則延遲時間 = 4秒 × 5 + 5秒 = 25秒
  4. 自動執行: 按順序打開每個鏈接，等待指定時間後移動文件到完成資料夾

資料夾結構

目標資料夾/
├── file1.html
├── file2.html
└── ...

處理後：

目標資料夾/
├── 00_finish_folder/
│   ├── file1.html
│   ├── file2.html
│   └── ...

延遲時間計算

  • 基礎延遲：每個鏈接 4 秒
  • 額外延遲：根據資料夾名稱中的數字範圍中點計算
  • 總延遲 = 基礎延遲 × 中點值 + 5秒

例如：

  • 資料夾名 0-9：中點 = 5，延遲 = 4 × 5 + 5 = 25秒
  • 資料夾名 10-20：中點 = 15，延遲 = 4 × 15 + 5 = 65秒

輸出信息

程序會顯示：

  • 預計總執行時間
  • 當前時間
  • 預計完成時間
  • 每個鏈接的處理狀態

注意事項

  1. 確保目標資料夾中包含有效的 HTML 文件
  2. HTML 文件必須包含符合 li.classroom-toc-item > a 選擇器的元素
  3. 程序會自動創建 00_finish_folder 資料夾來存放已處理的文件
  4. 程序使用單線程執行，確保鏈接按順序打開

錯誤處理

  • 如果資料夾名稱不包含有效的數字範圍格式，程序會拋出錯誤
  • 如果 HTML 文件中沒有找到符合條件的鏈接，該文件會被跳過
  • 如果目標資料夾不存在，程序會自動創建完成資料夾

自定義配置

您可以修改 app.py 中的以下參數：

  • self.eachWaitSecond = 4: 基礎延遲時間（秒）
  • 延遲計算公式：self.delay = self.eachWaitSecond * waitNumber + 5
