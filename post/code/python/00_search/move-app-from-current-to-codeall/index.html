應用程式移動工具 (Move App to CodeAll)

這是一個用於將應用程式從當前位置移動到 codeAll 目錄並建立雙向捷徑的 Python 工具。

功能說明

  • 將指定的資料夾移動到 codeAll 目錄結構中
  • 在原始位置建立指向新位置的捷徑
  • 在新位置建立指向原始位置的捷徑
  • 自動建立版本控制資料夾 (00_version)
  • 支援多種目標根目錄配置

系統需求

  • Windows 作業系統
  • Python 3.x
  • pywin32 套件 (用於建立 Windows 捷徑)

安裝依賴

pip install pywin32

使用方法

方法一：使用批次檔 (推薦)

go.bat "C:\path\to\your\app\folder"

方法二：直接使用 Python

python app.py "C:\path\to\your\app\folder"

支援的目標類型

目前支援以下目標名稱：

  • app - 移動到 E:\codeAll\python\00_tmp-not-organize
  • DIYDIY - 移動到 D:\00_DIY\kind\00_tmp-not-organize

工作流程

  1. 移動資料夾：將指定的資料夾移動到 codeAll 目錄結構中
  2. 建立前向捷徑：在原始位置建立指向新位置的捷徑 (格式：{資料夾名稱}___codeAll___.lnk)
  3. 建立版本資料夾：在新位置建立 00_version 資料夾
  4. 建立後向捷徑：在 00_version 資料夾中建立指向原始位置的捷徑 (格式：{資料夾名稱}___original___.lnk)

目錄結構範例

移動前：

C:\MyProject\
└── app\
    ├── main.py
    └── config.json

移動後：

E:\codeAll\python\00_tmp-not-organize\MyProject\
└── app\
    ├── main.py
    ├── config.json
    └── 00_version\
        └── app___original___.lnk

C:\MyProject\
└── app___codeAll___.lnk

錯誤處理

  • 如果目標資料夾已存在，程式會顯示錯誤訊息並退出
  • 如果指定的目標名稱不支援，程式會顯示支援的目標名稱清單
  • 如果命令列參數不正確，程式會顯示使用說明

注意事項

  • 請確保目標路徑存在且有寫入權限
  • 移動操作是不可逆的，請在執行前備份重要資料
  • 捷徑檔案使用 .lnk 副檔名，僅在 Windows 系統上有效

開發者資訊

這個工具專為管理 codeAll 目錄結構而設計，幫助開發者將應用程式整理到統一的目錄結構中，同時保持對原始位置的快速存取。
