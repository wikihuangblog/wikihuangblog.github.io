Wish Case OOP - 志願選填系統

這是一個基於 Python 和 SQLite 的志願選填系統，採用物件導向程式設計（OOP）架構重新開發。

📋 專案概述

Wish Case OOP 是一個完整的志願選填系統，提供學生登入、資料查詢、學校科系資訊管理等功能。系統採用現代化的 Web 介面設計，支援響應式布局和直觀的用戶體驗。

✨ 功能特色

🔐 用戶認證系統

  • 安全的用戶登入機制
  • 密碼加密存儲
  • 會話管理
  • 多角色支援（管理員、學生）

👨‍🎓 學生資料管理

  • 學生基本資料查詢
  • 身分證號驗證
  • 考試類別管理
  • 學校代碼關聯

🏫 學校科系資料

  • 完整的學校科系代碼資料庫
  • 考試類別分類
  • 動態查詢功能
  • API 端點支援

🌐 Web 介面

  • 現代化響應式設計
  • 直觀的用戶介面
  • 即時資料顯示
  • 移動設備友好

🗄️ 資料庫結構

系統包含四個主要資料表：

資料表 說明 記錄數
SCHCODE 學校科系代碼資料 ~239
EXAM_BASIC_STUDENT 學生基本資料 ~80
TITLE 考試類別資料 ~18
users 用戶登入資料 4

🚀 快速開始

1. 環境需求

  • Python 3.7+
  • SQLite 3
  • 現代瀏覽器

2. 安裝依賴

pip install -r wishcase-oop/requirements.txt

3. 設定資料庫

cd wishcase-oop
python initDB/setup_database.py

這將會：

  • 創建 SQLite 資料庫 (initDB/wishcase.db)
  • 從 initDB/orisql/ 目錄中的 SQL 檔案匯入資料
  • 創建測試用戶帳號

4. 測試系統

python test_system.py

5. 啟動 Web 服務器

python web_login.py

然後在瀏覽器中開啟：http://localhost:5000

🔌 API 端點

系統提供以下 RESTful API 端點：

學生資料

  • GET /api/student/<student_id> - 取得指定學生資料

學校科系

  • GET /api/schools?title_no=<title_no> - 取得指定考試類別的學校科系資料

考試類別

  • GET /api/titles - 取得所有考試類別資料

🛠️ 開發說明

核心架構

LoginSystem 類別

libs/login_system.py 提供：

  • 用戶認證 (login())
  • 學生資料查詢 (get_student_info())
  • 學校科系資料查詢 (get_school_codes())
  • 考試類別資料查詢 (get_titles())
  • 密碼雜湊 (hash_password())

Web 應用程式

web_login.py 提供：

  • Flask 路由處理
  • 會話管理
  • HTML 模板渲染
  • API 端點實作

資料庫設計

資料表關係

  • users 表關聯 EXAM_BASIC_STUDENT 表透過 student_id
  • EXAM_BASIC_STUDENT 表關聯 TITLE 表透過 EXAM_TITLE_NO
  • SCHCODE 表關聯 TITLE 表透過 TITLE_NO

資料匯入流程

  1. 解析 SQL 檔案中的 CREATE TABLE 語句
  2. 創建對應的 SQLite 資料表
  3. 解析 INSERT 語句並匯入資料
  4. 創建測試用戶帳號
