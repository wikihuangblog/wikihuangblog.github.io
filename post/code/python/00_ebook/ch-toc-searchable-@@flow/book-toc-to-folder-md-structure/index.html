書籍目錄轉資料夾結構工具

這是一個 Python 工具，可以將 Markdown 格式的書籍目錄（TOC）轉換成結構化的資料夾和檔案系統。

功能特色

  • 📁 自動建立階層式資料夾結構
  • 📝 為每個章節建立對應的 Markdown 檔案
  • 🔍 自動生成 Google 搜尋連結
  • 📚 支援以原始書籍名稱作為父資料夾
  • 🔢 自動格式化檔案名稱以利排序
  • 🗂️ 智慧整理章節資料夾結構

安裝需求

  • Python 3.6 或更高版本
  • 無需額外套件依賴

使用方法

基本用法

python app.py "書籍檔案名稱.md"

範例

python app.py "從零開始打造線上志願選填系統_前端-後端到雲端部署的完整實戰.md"

輸入格式

您的 Markdown 檔案應該包含階層式目錄結構，例如：

- **第 1 章：導讀與環境建置**
  - **1.1 本書導讀**
    - 本書適合誰？
    - 我們將一起打造出什麼系統？
  - **1.2 系統核心觀念與流程分析**
    - 什麼是「志願報名系統」？
    - 系統核心流程圖解
- **第 2 章：開發環境建置與技術棧選擇**
  - **2.1 打造現代化開發環境**
  - **2.2 技術棧選擇與介紹**

輸出結構

程式會建立以下資料夾結構：

00_output/
└── 書籍名稱/
    ├── 第 1 章：導讀與環境建置/
    │   ├── 第 01 章：導讀與環境建置.md
    │   ├── 1.1 本書導讀/
    │   │   └── 01.01 本書導讀.md
    │   └── 1.2 系統核心觀念與流程分析/
    │       └── 01.02 系統核心觀念與流程分析.md
    └── 第 2 章：開發環境建置與技術棧選擇/
        ├── 第 02 章：開發環境建置與技術棧選擇.md
        ├── 2.1 打造現代化開發環境/
        │   └── 02.01 打造現代化開發環境.md
        └── 2.2 技術棧選擇與介紹/
            └── 02.02 技術棧選擇與介紹.md

生成的檔案內容

每個生成的 Markdown 檔案包含：

# 章節標題

相關連結: https://www.google.com/search?q=章節標題

原始書籍名稱: 書籍檔案名稱

主要功能說明

1. 資料夾結構建立

  • 根據目錄層級自動建立對應的資料夾
  • 清理非法字元以確保檔案系統相容性

2. 檔案名稱格式化

  • 自動為個位數章節編號補零（例如：1 → 01）
  • 確保檔案按正確順序排列

3. 章節整理

  • 智慧識別主章節和子章節
  • 自動將子章節移動到對應的主章節資料夾內

4. 搜尋連結生成

  • 為每個章節自動生成 Google 搜尋連結
  • 清理標題中的特殊字元以優化搜尋

檔案說明

  • app.py - 主程式檔案
  • README.md - 專案說明文件

注意事項

  1. 確保輸入的 Markdown 檔案使用 UTF-8 編碼
  2. 程式會自動建立 00_output 資料夾作為輸出目錄
  3. 如果目標資料夾已存在，程式會覆蓋現有檔案
  4. 支援中文檔案名稱和資料夾名稱

錯誤處理

程式包含完整的錯誤處理機制：

  • 檔案讀取錯誤
  • 資料夾建立失敗
  • 檔案重新命名衝突
  • 路徑無效等情況
