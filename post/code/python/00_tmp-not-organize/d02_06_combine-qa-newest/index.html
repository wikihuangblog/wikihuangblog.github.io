金融科技問答合併工具

這是一個用於處理金融科技考試題目和答案的 Python 工具，能夠自動將分離的題目和答案文件合併成統一的格式。

專案概述

本專案主要用於處理金融科技相關的考試題目，包含：

  • 題目提取和解析
  • 答案格式識別和處理
  • 問答內容合併
  • 多種答案格式支援

檔案結構

app/
├── app.py                    # 主要程式檔案
├── Q.md                      # 題目檔案
├── A.md                      # 答案檔案
├── combined.md               # 合併後的輸出檔案
├── 00_version/               # 版本控制資料夾
│   ├── v1_combine-ok.py     # 第一版合併程式
│   └── v2_different-A-ok.py # 第二版（支援不同答案格式）
├── A-diff/                   # 答案差異比較
│   ├── Av1.md
│   └── Av2.md
├── Q-diff/                   # 題目差異比較
│   ├── v1.md
│   └── v2.md
└── README.md                 # 本說明檔案

功能特色

1. 智能題目解析

  • 自動識別題目編號和標題
  • 支援多種題目格式
  • 提取完整的題目內容

2. 多格式答案支援

  • 格式一：### 題目數字： 格式
  • 格式二：### **數字. 標題** 格式
  • 自動檢測並適配不同格式

3. 內容合併

  • 將題目和對應答案自動配對
  • 生成統一的 Markdown 格式輸出
  • 保持原始格式和連結

使用方法

基本使用

  1. 確保 Q.md 和 A.md 檔案存在於同一目錄
  2. 執行主程式：
python app.py
  1. 程式會自動生成 combined.md 檔案

程式流程

  1. 讀取檔案：讀取 Q.md 和 A.md
  2. 提取題目：使用正則表達式解析題目內容
  3. 提取答案：智能識別答案格式並提取
  4. 合併內容：將題目和答案配對
  5. 輸出結果：生成格式化的 combined.md

支援的題目格式

題目格式範例

[21.世界經濟論壇 WEF(2015)指出，金融科技可能導致下列何種關係的轉變？](https://app.yamol.tw/exam/107084)

(A) 通過金融機構與「同業」結合，將更難以滿足顧客金融需求
(B) 通過金融機構與「異業」結合，將更難以滿足顧客金融需求
(C) 顧客對「金融機構關係主導權」的瓦解
(D) 金融機構對「顧客關係主導權」的瓦解

答案：[D](https://app.yamol.tw/item/2897573)

答案格式範例

格式一：

### 題目21：金融科技對關係的轉變
- **題目**：世界經濟論壇WEF(2015)指出...
- **例子**：以前銀行是金融服務的主導者...

格式二：

### **21. 金融科技對關係的轉變**
詳細答案內容...

技術細節

正則表達式模式

題目提取：

pattern = r'\[(\d+)[\.\s](.*?)\]\s*\(https://app\.yamol\.tw/exam/\d+\)\s*(.*?)(?=\[\d+[\.\s]|$)'

答案提取（格式一）：

pattern1 = r'### 題目(\d+)：(.*?)(?=### 題目|\Z)'

答案提取（格式二）：

pattern2 = r'### \*\*(\d+)\.(.*?)\*\*(.*?)(?=### \*\*|\Z)'

輸出格式

合併後的檔案會包含：

  • 題目標題和編號
  • 完整的題目內容
  • 對應的答案內容
  • 分隔線用於區分不同題目

注意事項

  1. 確保輸入檔案使用 UTF-8 編碼
  2. 題目和答案的編號必須對應
  3. 程式會自動跳過沒有對應答案的題目
  4. 建議在執行前備份原始檔案

系統需求

  • Python 3.6+
  • 無需額外依賴套件
