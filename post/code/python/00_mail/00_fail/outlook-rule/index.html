Outlook 規則創建工具

這個項目包含多個腳本來解決 Outlook 規則創建時的參數錯誤問題。

問題分析

根據測試結果，主要問題是：

  • 主題條件設置失敗：(-2147352571, '類型不符。', None, 1)
  • 規則保存失敗：(-2147352567, '發生例外狀況。', (4096, 'Microsoft Outlook', '由於無效的動作或條件，無法儲存一個或多個規則。', None, 0, -2147467259), None)
  • 規則衝突：(-2147352567, '發生例外狀況。', (4096, 'Microsoft Outlook', '一個或多個規則發生衝突。您必須解決衝突後，才能儲存規則。', None, 0, -2147352567), None)

這表明 Outlook 的 COM 接口對主題條件的設置方式有特定要求，並且可能存在規則衝突。

解決方案

1. 環境測試

首先運行環境測試腳本來檢查您的 Outlook 環境：

python test_outlook.py

2. 清理版本（推薦）

使用清理版本，先刪除所有相關規則，然後重新創建：

python app_clean.py

這個版本會清理所有相關的規則，避免衝突。

3. 唯一名稱版本

使用時間戳生成唯一規則名稱，避免衝突：

python app_unique.py

這個版本使用唯一的規則名稱，避免與現有規則衝突。

4. 修復版本

使用修復版本，嘗試多種主題條件設置方法：

python app_fixed.py

5. 無條件版本

如果主題條件仍然有問題，使用無條件版本：

python app_no_condition.py

6. 其他版本

  • app_sender_condition.py - 發件人條件版本
  • app_simple.py - 簡化版本
  • app_advanced.py - 進階版本
  • app.py - 修改後的原始版本

測試結果分析

根據您的測試結果：

  • ✅ Outlook 連接正常
  • ✅ 文件夾創建正常
  • ✅ 規則創建正常
  • ✅ 移動動作設置正常
  • ❌ 主題條件設置失敗
  • ❌ 規則保存失敗（由於無效條件或規則衝突）

建議的使用順序

  1. 先運行清理版本：python app_clean.py
  2. 如果失敗，使用唯一名稱版本：python app_unique.py
  3. 手動在 Outlook 中編輯規則，添加主題條件

手動編輯規則步驟

如果腳本創建了無條件規則，您可以手動在 Outlook 中編輯：

  1. 打開 Outlook
  2. 點擊「檔案」→「管理規則和通知」
  3. 找到創建的規則（如 Aomei_Rule_20250101_120000）
  4. 點擊「編輯」
  5. 添加條件：「主題包含特定文字」
  6. 輸入「aomei」
  7. 保存規則

規則衝突解決方案

如果遇到規則衝突錯誤：

  1. 手動清理：

    • 打開 Outlook
    • 檔案 → 管理規則和通知
    • 刪除所有相關的規則
    • 重新運行腳本

  2. 使用清理腳本：

    python app_clean.py

  3. 使用唯一名稱：

    python app_unique.py

常見問題

Q: 為什麼會出現規則衝突？

A: 這通常是因為：

  • 已存在同名規則
  • 規則設置有衝突
  • Outlook 規則管理器的限制

Q: 為什麼主題條件設置會失敗？

A: 這可能是因為：

  • Outlook 版本差異
  • 主題條件需要特定的格式
  • COM 接口的限制

Q: 無條件規則會移動所有郵件嗎？

A: 是的，無條件規則會移動所有收到的郵件。建議：

  1. 先創建無條件規則
  2. 立即在 Outlook 中手動添加條件
  3. 或者暫時禁用規則，直到添加條件

Q: 如何確保規則正確工作？

A:

  • 運行測試腳本檢查環境
  • 在 Outlook 中手動驗證規則
  • 發送測試郵件驗證功能

PowerShell 版本

項目還包含一個 PowerShell 腳本 create_outlook_rule.ps1，可以作為 Python 腳本的替代方案。
