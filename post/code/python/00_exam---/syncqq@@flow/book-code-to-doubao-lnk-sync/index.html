程式碼同步工具 (Code Sync Tool)

專案概述

這是一個用於處理程式碼文件的 Python 工具，主要功能是將指定目錄下的程式碼文件（如 .java、.py 等）進行結構化整理，為每個程式碼文件創建對應的目錄和 Markdown 文檔。

主要功能

  • 🔄 自動化處理：批量處理指定目錄下的程式碼文件
  • 📁 目錄結構化：為每個程式碼文件創建對應的目錄
  • 📝 文檔生成：自動生成 .code.md 文件，包含原始程式碼內容
  • ⏰ 時間戳保持：保持原始文件的創建和修改時間
  • 🛡️ 錯誤處理：完善的異常處理和編碼錯誤處理
  • 🔍 深度限制：支持目錄深度限制，避免過度遞歸

系統需求

  • Python 3.6+
  • Windows 系統（使用 Windows API 設置文件時間）
  • 必要的 Python 套件：
    • pywin32 (用於 Windows API 操作)

安裝依賴

pip install pywin32

使用方法

基本用法

python app.py <目標目錄路徑>

範例

# 處理當前目錄下的 target 文件夾
python app.py target

# 處理指定路徑的目錄
python app.py "C:\path\to\your\code\directory"

處理流程

  1. 掃描文件：遞歸掃描目標目錄下的所有 .java 文件
  2. 創建目錄：為每個程式碼文件創建以文件名命名的目錄
  3. 生成文檔：創建 .code.md 文件，包含原始程式碼內容
  4. 移動文件：將原始程式碼文件移動到新創建的目錄中
  5. 保持時間戳：使用 Windows API 保持原始文件的創建和修改時間

目錄結構

處理前的目錄結構：

target/
├── 006_下一個更大的數.java
├── 007_字符串中的單詞數.java
└── other_code.java

處理後的目錄結構：

target/
├── 006_下一個更大的數/
│   ├── 006_下一個更大的數.java
│   └── 006_下一個更大的數__________{}______.code.md
├── 007_字符串中的單詞數/
│   ├── 007_字符串中的單詞數.java
│   └── 007_字符串中的單詞數__________{}______.code.md
└── other_code/
    ├── other_code.java
    └── other_code__________{}______.code.md

版本歷史

本專案包含多個版本迭代：

  • v1: 基礎功能實現
  • v2: 支持遞歸處理和深度限制
  • v3: 添加文件時間保持功能
  • v4: 解決字符編碼問題
  • v5: 最終穩定版本，完善錯誤處理

注意事項

  1. 權限要求：確保對目標目錄有讀寫權限
  2. 備份建議：運行前建議備份重要文件
  3. 編碼支持：支持 UTF-8 編碼的程式碼文件
  4. 深度限制：默認限制處理深度為 2 層，避免過度遞歸
  5. 重複處理：已處理過的目錄會被自動跳過

錯誤處理

工具包含完善的錯誤處理機制：

  • 編碼錯誤：自動跳過無法讀取的文件
  • 權限錯誤：提示權限不足的文件
  • 路徑錯誤：處理無效的文件路徑
  • 時間設置錯誤：警告無法設置時間戳的文件

開發說明

核心函數

  • set_file_times(): 使用 Windows API 設置文件時間戳
  • process_code_files(): 主要的文件處理邏輯

自定義配置

可以通過修改代碼中的以下參數來自定義行為：

  • 文件擴展名過濾（目前支持 .java）
  • 目錄深度限制
  • 輸出文件命名格式

授權

本專案僅供學習和個人使用。
