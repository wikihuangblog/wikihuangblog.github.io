Cursor 聊天歷史提取器

這是一個用於提取和整理 Cursor 編輯器聊天歷史記錄的 Python 工具。該工具可以從 Cursor 的本地數據庫中提取聊天記錄，並將其整理成易於閱讀的 Markdown 格式文件。

功能特點

  • 🔍 自動搜索: 自動在 Cursor 的 workspaceStorage 目錄中搜索聊天數據庫
  • 📊 數據提取: 從 SQLite 數據庫中提取聊天記錄數據
  • 📝 格式化輸出: 將聊天記錄轉換為結構化的 Markdown 格式
  • 📁 智能組織: 按日期和時間自動組織輸出文件夾
  • 🔗 工作區鏈接: 在輸出文件中包含原始工作區路徑的鏈接
  • 📋 日誌記錄: 完整的操作日誌記錄，便於調試和監控

項目結構

app/
├── main.py                    # 主程序入口
├── config/
│   └── logging_config.py     # 日誌配置
├── libs/
│   ├── chat_processor.py     # 聊天記錄處理器
│   ├── database_handler.py   # 數據庫操作處理器
│   ├── file_handler.py       # 文件操作處理器
│   └── folder_organizer.py   # 文件夾組織器
├── logs/                     # 日誌文件目錄
├── exportAUTO/              # 自動導出功能
└── cursor-history.bat       # Windows 批處理啟動腳本

安裝要求

  • Python 3.6+
  • 無需額外依賴包（使用 Python 標準庫）

使用方法

方法一：使用批處理文件（推薦）

# 雙擊運行
cursor-history.bat

方法二：直接運行 Python 腳本

python main.py

配置說明

在 main.py 中可以修改以下路徑配置：

# Cursor 工作區存儲路徑
root_path = r"C:\Users\zero\AppData\Roaming\Cursor\User\workspaceStorage"

# 處理後文件移動目標路徑
destination_path = r"C:\Users\zero\AppData\Roaming\Cursor\User\koo"

# 聊天記錄輸出基礎路徑
output_base_path = r"D:\diary\2025\03\00_AIQA\00_cursor\ChatHistories"

工作流程

  1. 搜索數據庫: 在指定的根路徑下搜索 state.vscdb 文件
  2. 提取數據: 從 SQLite 數據庫中提取聊天記錄 JSON 數據
  3. 處理聊天: 解析聊天記錄，按時間排序
  4. 生成文件名: 根據聊天內容自動生成有意義的文件名
  5. 保存文件: 將聊天記錄保存為 Markdown 格式
  6. 組織文件夾: 按日期和時間重新組織輸出文件夾結構
  7. 清理: 移動已處理的原始文件夾

輸出格式

生成的 Markdown 文件包含：

  • 工作區路徑鏈接
  • 按時間排序的聊天記錄
  • 發送者標識（User/AI）
  • 時間戳
  • 消息內容

示例輸出：

[Workspace Path](file:///path/to/workspace)

---

[2025-01-15 10:30:15] User:
請幫我寫一個 Python 函數

[2025-01-15 10:30:45] AI:
好的，我可以幫您寫一個 Python 函數...

日誌功能

  • 詳細的操作日誌記錄在 logs/ 目錄下
  • 按日期分割日誌文件
  • 支持控制台和文件雙重輸出
  • 自動日誌輪轉（10MB 大小限制，保留 5 個備份）

文件夾組織

輸出文件夾按以下結構組織：

ChatHistories/
├── 2025/
│   ├── 01/
│   │   ├── 15/
│   │   │   ├── 103015/
│   │   │   │   └── chat_record.md
│   │   │   └── 143022/
│   │   │       └── another_chat.md

注意事項

  • 確保有足夠的磁盤空間用於存儲提取的聊天記錄
  • 程序會自動創建必要的目錄結構
  • 如果目標文件夾已存在，會自動生成唯一名稱
  • 建議在運行前備份重要的聊天數據

故障排除

  1. 權限問題: 確保程序有讀取 Cursor 數據庫和寫入輸出目錄的權限
  2. 路徑不存在: 檢查配置的路徑是否正確存在
  3. 數據庫損壞: 如果遇到數據庫讀取錯誤，可能是數據庫文件損壞
  4. 查看日誌: 檢查 logs/ 目錄下的日誌文件獲取詳細錯誤信息

開發說明

核心類說明

  • DatabaseHandler: 負責 SQLite 數據庫操作和數據提取
  • FileHandler: 處理文件搜索、創建和路徑管理
  • ChatProcessor: 聊天記錄的解析、格式化和輸出
  • FolderOrganizer: 輸出文件夾的重新組織和整理

擴展功能

可以通過修改以下文件來擴展功能：

  • 添加新的輸出格式支持
  • 實現聊天記錄的過濾和搜索
  • 添加數據統計和分析功能
