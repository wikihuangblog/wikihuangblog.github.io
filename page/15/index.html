outlook-rule

Outlook 規則創建工具

這個項目包含多個腳本來解決 Outlook 規則創建時的參數錯誤問題。

問題分析

根據測試結果，主要問題是：

  • 主題條件設置失敗：(-2147352571, '類型不符。', None, 1)
  • 規則保存失敗：(-2147352567, '發生例外狀況。', (4096, 'Microsoft Outlook', '由於無效的動作或條件，無法儲存一個或多個規則。', None, 0, -2147467259), None)
  • 規則衝突：(-2147352567, '發生例外狀況。', (4096, 'Microsoft Outlook', '一個或多個規則發生衝突。您必須解決衝突後，才能儲存規則。', None, 0, -2147352567), None)

這表明 Outlook 的 COM 接口對主題條件的設置方式有特定要求，並且可能存在規則衝突。

split-big-toc-into-each-chapter

電子書目錄分割工具

這是一個用於處理電子書目錄的 Python 工具，可以將大型的目錄文件自動分割成各個章節的獨立 Markdown 文件，並進行格式化處理。

功能特色

  • 📚 自動章節分割：將大型目錄文件按章節自動分割
  • 🔧 智能格式化：自動處理標題格式、編號和空格
  • 📁 結構化組織：為每個章節創建獨立的資料夾和文件
  • 🏷️ 標準化命名：使用統一的章節編號格式（ch01, ch02…）
  • 🔄 批量處理：一次性處理整個目錄結構

項目結構

app/
├── app.py                 # 主程序文件
├── toc.md                # 原始目錄文件
├── toc_modified.md       # 格式化後的目錄文件
├── chapter/              # 章節文件輸出目錄
├── 00_issue/            # 問題記錄目錄
│   └── issue.md
├── ref/                 # 參考文件目錄
│   └── format-toc.py
└── run.bat              # Windows 批次執行文件

安裝與使用

系統需求

  • Python 3.6 或更高版本
  • Windows 系統（支援批次文件執行）

使用方法

方法一：使用批次文件（推薦）

# 在 Windows 命令提示字元中執行
run.bat toc.md

方法二：直接執行 Python 腳本

# 在專案根目錄執行
python app.py toc.md

參數說明

  • toc.md：輸入的目錄文件路徑（必需參數）

處理流程

  1. 讀取輸入文件：載入指定的目錄文件
  2. 格式化處理：
    • 將中文括號 （） 轉換為英文括號 ()
    • 處理標題層級和編號格式
    • 統一章節編號格式為 ch01_, ch02_ 等
  3. 生成格式化文件：保存為 toc_modified.md
  4. 分割章節：
    • 按 ### 標記分割章節
    • 為每個章節創建獨立資料夾
    • 生成對應的 Markdown 文件
